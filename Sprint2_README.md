Sprint 3 MVP Scope (Planned)

The minimum required database-driven pages for Sprint 3 implementation are:

/users – Users list page

/users/:id – User profile page

/items – Listings page

/items/:id – Item detail page

/tags – Tags/Categories page

/tags/:id/items – Items filtered by tag

These pages will dynamically retrieve data from the MySQL database and render content using Express and PUG templates.

User Stories (Sprint 3 Selected)
US-01: Browse Items/Listings (Must)

As a student,
I want to browse available items,
So that I can find resources I need.

US-02: Search/Filter Items (Should)

As a student,
I want to filter items by tags or categories,
So that I can quickly find relevant resources.

US-03: View Item Detail (Must)

As a student,
I want to view detailed information about an item,
So that I can decide whether to request it.

US-04: Browse Tags/Categories (Must)

As a student,
I want to browse items grouped by tags,
So that I can explore resources by type.

US-05: Browse Users + View Profile (Must)

As a student,
I want to view user profiles,
So that I can see who posted an item.

US-06: Safety Guidance Visible (Must)

As a student,
I want safety guidance displayed clearly on the platform,
So that exchanges can happen responsibly.
